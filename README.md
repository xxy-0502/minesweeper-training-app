# 扫雷康复训练游戏系统

一个面向课程设计的扫雷训练 Web 应用。项目基于 React、Vite 和 TypeScript 开发，围绕“配置训练、完成扫雷操作、生成训练报告、查看历史水平”形成完整训练闭环。

## 功能概览

- 支持配置训练时长和训练难度。
- 进入训练前提供 `3、2、1、Start` 准备倒计时，准备时间不计入正式训练数据。
- 支持左键翻开方块，右键或长按插旗。
- 支持暂停、继续、提前结束并查看报告。
- 单局胜利或失败后短暂停留，并显示自动进入下一局的倒计时提醒。
- 训练结束后生成报告，展示分数、总操作次数、正确数量、错误数量和平均反应时间。
- 报告页使用 ECharts 展示每秒完成次数、正确/错误对比和历史水平趋势。
- 最近一次报告和历史报告会保存在浏览器 `localStorage` 中，刷新页面后仍可查看。

## 统计口径

- 总题数/总次数：训练中的有效操作次数，来自 `stats.totalActions`。
- 正确数量：安全翻开数量加结束时正确插旗数量。
- 错误数量：踩雷次数加结束时错误插旗数量。
- 每秒完成次数：正式训练开始后，每一秒内完成的有效操作数量。
- 平均反应时间：相邻有效操作之间的平均时间间隔。
- 分数：根据胜利、安全翻开、正确插旗和错误数量综合计算。

为了避免通过反复右键插旗即时判断是否为雷，插旗操作不会立即改变正确/错误插旗分数。系统会在训练结束生成报告时扫描最终棋盘状态，再统一计算正确插旗、错误插旗和最终分数。

## 本地运行

请先安装 Node.js 和 pnpm，然后在项目根目录执行：

```bash
pnpm install
pnpm dev
```

开发服务器启动后，终端会显示访问地址，通常是：

```text
http://localhost:5173/
```

如果在 VSCode 中运行，建议先打开项目目录：

```text
D:\CodeProject\saolei
```

然后在 VSCode 终端中执行启动命令。

## 常用命令

```bash
pnpm dev
pnpm build
pnpm lint
pnpm format
pnpm preview
```

## 目录结构

```text
src/
  components/   通用组件、棋盘组件、报告图表组件
  data/         难度等静态数据
  game/         扫雷棋盘、规则和统计逻辑
  pages/        配置页、游戏页、报告页
  store/        训练配置和游戏状态管理
  types/        业务类型定义
  utils/        报告本地存储工具
```

## 页面流程

1. 在配置页选择训练时长和难度。
2. 点击开始训练，等待准备倒计时结束。
3. 在正式训练中进行翻开、插旗、暂停或提前结束。
4. 时间结束或手动结束后进入报告页。
5. 在报告页查看本次训练图表和历史水平趋势。

## 报告本地存储

报告数据存储在当前浏览器的 `localStorage` 中：

- `minesweeper-training-app/latest-report`：最近一次训练报告。
- `minesweeper-training-app/report-history`：历史训练报告，最多保留 30 条。

这些数据只保存在本地浏览器中，不会上传到服务器。
